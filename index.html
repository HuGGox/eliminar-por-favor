Las terminales son: Es una linea de comandos, nosotros a traves de estos podemos hacer ciertas acciones en windows. Podemos crear, eliminar, etc archivos, carpetas, etcs. El cmd no distingue de mayusculas y minusculas por ende podemos escribir con cualquiera de las 2 que no habria inconvenientes. Escribir help muestra todos los comandos en ESPAÃ‘OL. La diferencia entre cmd y power shell es que cmd permite los espacios, mientras que power shell requiere "". Con cls limpiamos el cmd. Poniendo cd y arrastrando la carpeta al cmd podemos entrar directamente sin tener que entrar carpeta por carpeta. Tambien podemos abrir la carpeta y donde tenemos la localizacion podemos poner cmd y automaticamente la abre en el cmd Mkdir o MD crean una carpeta en el directorio en el que estemos Con Move movemos el archivo a una carpeta, por ej move app.js assets (carpeta en la que queremos el archivo) Con cd .. volvemos para atras 1 carpeta. GIT: Con git help vemos todos los comandos que tiene disponible. Con git add . (el punto espaciado) agrega todos los archivos a git, con un nombre en especifico agrega 1 solo archivo. Con git status -s git empieza a hacer un seguimiento de todos los archivos que estan dentro de la carpeta. Con git commit -m "" dejamos un mensaje dentro de las comillas para etiquetar la carpeta Con git log --oneline vemos el nombre del repositorio Con git init empezamos nuestro repositorio, solamente se ejecuta 1 vez por proyecto. Si entramos en el modo vim por errar en un comando de git, apretamos :q! y salimos. Cuando estemos trabajando con un repositorio remoto, primero subimos los archivos a un nuevo repositorio de github, nos va a dar un "git remote add origin https://github.com/HuGGox/eliminar-por-favor.git", agregamos con git branch -m master, ponemos git push -u origin master y despues cada vez que tengamos que agregar alguna modificacion usamos git add ., git commit -m "" y por ultimo git push Con git clone y un codigo que nos va a dar GitHub podemos recuperar una carpeta que hayamos subido con los archivos que se hayan borrado desde la consola. Tenemos que crear una carpeta y ponerla en el cmd. Una vez subido nuestro repositorio, si vamos a settings, pages y le damos en master / main en la parte de branch, guardamos, esto nos va a dar como respuesta una pagina con nuestro repositorio.
